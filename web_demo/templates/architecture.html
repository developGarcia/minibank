Arquitectura C4 - Sistema MiniBank

Sistema implementado usando la metodología de modelado Arquitectura C4

Nivel 1 - Contexto

¿Qué hace el sistema?

Sistema bancario MiniBank que permite a los clientes gestionar sus cuentas y realizar transacciones.

Actores Principales:
  • Cliente: Realiza operaciones bancarias
  • Administrador: Gestiona el sistema
Relaciones:
  • Cliente → Consulta saldos
  • Cliente → Realiza transacciones
  • Administrador → Supervisa sistema
Nivel 2 - Contenedores

¿De qué partes está compuesto?

API Gateway
Puerto 5000

Punto único de entrada, enrutamiento de requests

Flask
Auth Service
Puerto 5001

Autenticación, autorización y gestión de tokens

Flask
Account Service
Puerto 5002

Gestión de cuentas, transacciones y saldos

Flask
Nivel 3 - Componentes

¿Cómo está organizado internamente cada contenedor?

API Gateway
  • GatewayManager: Enrutamiento
  • HealthChecker: Monitoreo
  • RequestRouter: Distribución
Auth Service
  • AuthenticationManager: Lógica de auth
  • Login Endpoint: Recepción credenciales
  • Token Verification: Validación tokens
Account Service
  • AccountManager: Lógica de cuentas
  • Transaction Engine: Procesamiento
  • Account Endpoints: Operaciones CRUD

Flujos de Negocio:
  1. Autenticación:
    Cliente → Gateway → Auth Service → Token
  2. Consulta:
    Cliente → Gateway → Account Service → Datos
  3. Transacción:
    Cliente → Gateway → Account Service → Procesamiento
  4. Validación:
    Verificación de fondos → Actualización → Historial
